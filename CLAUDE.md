# Deloros Bot — инструкции для Claude Code

Бот для чата сообщества «Деловая Россия» в мессенджере **MAX**. Архитектурно — копия `valentin_bot` (тот же стек ядра и паттерны), но транспорт **MAX (`maxapi`)** вместо Telegram (aiogram). Назначение своё: держать общий контекст сообщества (кто есть кто, кто чем полезен, метчинг спроса и предложения) и онбордить участников через интервью.

## Обязательное правило
При любом добавлении или изменении инструментов агента (`tools/`, `agent.py`) **всегда обновлять `SYSTEM_PROMPT` в `agent.py`** — чтобы бот знал о своих возможностях.

## Назначение (замысел продукта)
Ассистент чата сообщества «Деловая Россия». Воплощает тезис проекта AI Transformation: AI держит **общий контекст** сообщества, который не удержит ни один участник, и **синтезирует** связи (кто кому полезен). Не заменяет нетворкинг — усиливает. Админ — Александр. Бот живёт в общем чате, доступен всем участникам.

### Возможности (по приоритету)
1. **Кто есть кто.** Профили участников (`member`): компетенции, чем полезен, что ищет. **Семантический** поиск «кто разбирается в X / кто поможет с Y» через `search_kb(members)` — по смыслу, а не по точному слову (эмбеддинги).
2. **Онбординг через интервью.** Новый участник не пишет о себе сам — бот опрашивает СТРОГО по одному вопросу и собирает профиль. Вопросы: чем занимаешься → компетенции/опыт → чем полезен сообществу → что сейчас ищешь → зачем пришёл в сообщество → интересы вне бизнеса/хобби. Контакт не спрашивается — берётся из MAX-аккаунта (username). Можно прислать резюме/биографию **файлом** (PDF/DOCX/TXT) или **голосом** — бот извлечёт профиль и переспросит только недостающее. На выходе `save_to_kb(member)`.
3. **Метчинг спроса/предложения.** Ловит «ищу …» и «могу предложить …», сводит людей: `search_kb(members)` + `search_chat_log`. Важные — в `request`. Проактивный метчинг (реакция без упоминания) пока не реализован — нужен и код, и права админа бота в группе.
4. **Общий ассистент.** Саммари обсуждений, веб-поиск, новости, напоминания, разбор документов и **картинок** (gpt-4o vision). Документ по упоминанию раскладывается по категории: резюме→`member`, протокол→`meeting`, исследование→`research`.

## Стек и транспорт (MAX / maxapi)
- Транспорт — `maxapi` (long-polling). API повторяет aiogram: `from maxapi import Bot, Dispatcher`, `Command`, `MessageCreated`. Референс рабочего бота — `AIMindsetNov/smm_monitor/max_bot.py`.
- Токен бота берётся у `@MasterBot` в MAX. Доки: dev.max.ru/docs.
- Ядро (`agent.py`, `tools/`) транспортно-независимо — принимает `chat_id, username, text`, возвращает текст; при миграции с Telegram не меняется.

## Архитектура
- `main.py` — запуск, инициализация MAX-бота, long-polling, `set_api_url` до первого запроса
- `handlers.py` — обработчики MAX (@упоминания, голос, документы, команды). В группе реагирует на текст И вложения только по упоминанию/reply; в личке — всегда
- `agent.py` — агентный цикл OpenAI (gpt-4o, function calling), `SYSTEM_PROMPT`, `TOOLS`
- `scheduler.py` — фоновая проверка напоминаний
- `tools/` — инструменты агента (включая `embeddings.py` — семантический поиск)
- `panel/` — админ-панель (FastAPI + Jinja2, сервис `deloros-panel`, стиль Steep, логотип). Вкладки: **Обзор** (метрики + блоки статистики использования: последняя активность, активность за 30 дней со спарклайном, охват/вовлечённость, лента обращений — `panel.data.usage_overview()`) / **Реестр** (CRUD: добавить/удалить/полное редактирование карточки всех полей `/roster/edit`, права админа; сортировка по ФИО; председатель `CHAIRMAN_PHONE` закреплён первым с бейджем) / **Документы** / **О боте** + страница участника (`/member/{phone}`: профиль, расходы, документы, активность). Вход **по телефону+паролю** для участников с галочкой «админ» (`tools/admins.py`, хеш pbkdf2, `admins.json` вне git). `panel/data.py` агрегирует расходы (usage.jsonl), документы, активность (chat_logs) по участнику. Доступ снаружи — только через делорос.рф (NPM); напрямую порт 8087 из корп-сети.

### Доступ по телефону (Фаза 1)
- В **личке** доступ только участникам клуба. Заход по ссылке/`/start` → бот просит «📞 Поделиться номером» (`RequestContactButton`).
- Телефон из CONTACT-вложения (`payload.vcf.phone`) нормализуется и сверяется с **реестром** `knowledge_base/roster.md` (`tools/roster.py`). Совпал → `tools/access.py` помечает `user_id` подтверждённым (`verified_users.json`), запускается онбординг. Не совпал → «телефона нет среди участников клуба».
- Неподтверждённый в личке — любое сообщение упирается в просьбу поделиться номером. **Группа гейтом по телефону не охвачена** (там доступ = членство в группе).
- `roster.md` (PII) и `verified_users.json` — вне git, `roster.md` самовосстанавливается из шаблона. Реестром пока управляют вручную; админ-панель — Фаза 2.

### Поведение в группе и видимость сообщений
- MAX по умолчанию в privacy-режиме: без прав админа бот получает в группе **только сообщения с @упоминанием**. Чтобы видел всё — сделать бота **админом чата** (настройка в MAX, не в коде).
- Код уже логирует весь полученный текст группы в `chat_log` (пассивно). С админкой лог наполнится всей историей автоматически.
- Гейт по упоминанию охватывает и вложения — чтобы с админкой бот не реагировал на каждый файл/голос в чате.

## Чем отличается от valentin_bot
- **Транспорт:** MAX (`maxapi`) вместо Telegram (aiogram).
- **Личность и роль:** «Делорос», ассистент сообщества; ядро ценности — общий контекст связей.
- **Категории KB** (изменены в `tools/kb_save.py`, `tools/kb_search.py`): `member`, `company`, `offer`, `request`, `meeting`, `research` (+ transcription/document). Главная новая — **`member`** (профиль участника).
- **Онбординг через интервью:** бот сам опрашивает участника по одному вопросу и сохраняет профиль, человек не пишет о себе сам (логика в `SYSTEM_PROMPT`).
- **Метчинг запросов:** ловит «ищу …» / «могу предложить …», сводит через `search_kb(members)` + `search_chat_log`.
- **Таймзона напоминаний:** Москва (UTC+3) — поменяна с Иркутска; подтвердить под филиал.

## Реестр (roster.md) — таблица участников
Markdown-таблица `knowledge_base/roster.md` (PII, вне git), колонки: `ФИО | Телефон | Дата рождения | Компания | Должность | Отрасль | Email | Добавлен`. Парсер (`tools/roster.py`) обратно совместим: ≥8 колонок — с email, 7 — без email, 3 — старый формат (доп. поля пустые). Функции: `load_roster/add_member/upsert_member/update_member/rename_member/delete_member`. Телефон-ключ доступа — мобильный. Смена телефона в панели переносит админ-права (`admins.migrate_phone`) и подтверждение (`access.migrate_phone`).

## Модель данных участника (member, профиль в KB)
Markdown в `knowledge_base/members/`, фронтматтер как в valentin (`type/name/added/tags/related`). Контент-профиль:
- Чем занимается / компания / роль
- Компетенции (теги для поиска: юрист, экспорт, логистика, финансы, GR, …)
- Чем может быть полезен сообществу
- Что сейчас ищет / запросы
- Зачем пришёл в сообщество (цели, мотивация)
- Интересы вне бизнеса, хобби
- Контакт — из MAX-аккаунта (username), не спрашивается

## Деплой
Тот же сервер (Ubuntu, корпоративная сеть), что и `valentin_bot`. Отдельный репозиторий, отдельный systemd-сервис `deloros-bot`, отдельная папка `/home/alex/deloros_bot`. Unit-файл лежит в репо — `deploy/deloros-bot.service`.

### Первичная установка на сервере (разово)
```bash
cd /home/alex
git clone https://github.com/happyalexandr-git/deloros-bot.git
cd deloros-bot
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
cp .env.example .env          # вписать MAX_TOKEN + OPENAI_API_KEY (+ TAVILY_API_KEY)
sudo cp deploy/deloros-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now deloros-bot
journalctl -u deloros-bot -f  # проверить запуск
```

### Админ-панель
Отдельный сервис `deloros-panel` (`deploy/deloros-panel.service`), uvicorn на **порту 8087** (8080 на srv001 занят другим сервисом; только корп-сеть). Доступ: `http://srv001:8087`. **Вход по телефону+паролю** для участников-админов (`tools/admins.py`, `admins.json` вне git), НЕ из `.env`. Бутстрап первого админа на сервере:
```bash

sudo cp deploy/deloros-panel.service /etc/systemd/system/
```

### Принятые решения
- **`handlers.py`** экспортирует `register_handlers(dp, bot, bot_id, bot_username)`; `main.py` берёт личность бота из `bot.get_me()`.
- **Endpoint MAX:** `platform-api.max.ru` (серт Let's Encrypt). `platform-api2.max.ru` требует российский УЦ Минцифры (нет в certifi) — НЕ используем. Переопределяется `MAX_API_URL`.
- **Онбординг:** триггер — `BotStarted` (нажатие «Начать») + `/start` + @упоминание.
- **Упоминание в группе:** `@username` в тексте, `USER_MENTION` в `body.markup` по `user_id` бота, либо reply на бота.
- **Голос:** работает ТОЛЬКО на webhook-транспорте — по long-poll MAX шлёт пустой `message_created` без аудио (подтверждено на HTTP-уровне и сравнением с vAptekeMaxBot). `_handle_audio`: если `transcription` есть — берём её; иначе скачиваем по `payload.url` и распознаём через Whisper (`tools/voice_transcribe.py`, `whisper-1`).
- **Webhook:** `MAX_TRANSPORT=webhook` в `.env` → `AiohttpMaxWebhook` (maxapi) на порту `MAX_WEBHOOK_PORT=8088`, публичный URL `MAX_WEBHOOK_URL`, секрет `MAX_WEBHOOK_SECRET` (MAX шлёт его в `X-Max-Bot-Api-Secret`, maxapi проверяет сам, чужие POST → 403). Подписка при старте: `delete_webhook` → `subscribe_webhook`. Режим polling остаётся дефолтом (для локальной отладки) и сам снимает webhook-подписку.
- **Документы:** скачиваются по `attachment.payload.url` через `httpx`; агент раскладывает по категории (member/meeting/research/резюме). Сырой текст всегда сохраняется как `document` для поиска.
- **Картинки:** ловим `AttachmentType.IMAGE`, скачиваем сами и шлём в gpt-4o vision **base64** (`tools/image_describe.py`) — НЕ по URL (OpenAI не всегда может скачать MAX-ссылку). Описание (что на картинке + извлечённый текст) уходит агенту как текст, подпись-вопрос фокусирует. `_handle_image` в handlers.py.
- **Учёт использования:** каждое обращение к агенту пишется в `usage.jsonl` через `log_usage` с полем `kind` (text/voice/image/document — параметр `run_agent(kind=...)`). `panel.data.usage_overview(roster)` строит блоки «Обзора» (таймзона Иркутск UTC+8); имя резолвится из `verified_users` по username.
- **Реестр как справочник:** `roster.md` — 8-колоночная таблица (см. раздел «Реестр»). Председатель закреплён первым через `CHAIRMAN_PHONE` в `panel/app.py`.
- **Поиск:** семантический (`tools/embeddings.py`, gpt-эмбеддинги), порог сходства 0.2, подстрочный фолбэк при сбое OpenAI.
- **KB вне git:** профили/INDEX/кэш векторов в `.gitignore` (приватность + чтобы рантайм-записи не конфликтовали с `git pull`). `INDEX.md` самовосстанавливается.
- **Вход в панель:** по телефону+паролю участников с галочкой «админ» (хеш pbkdf2 в `admins.json`, вне git). Не из `.env`. Бутстрап первого админа — `tools.admins.set_admin(phone, pw)`.
- **Уведомления:** `notify_participants` (только админ) → `add_reminder(..., targets=[user_id])` → `scheduler` шлёт в личку каждому. Адресат резолвится `agent._resolve_targets` (all / имя / телефон по реестру+`verified_users`).
- **Связка участника:** реестр (телефон) ↔ `verified_users.json` (`user_id`+`username`, пишется при верификации) ↔ профиль `members/<slug имени>.md` ↔ расходы/логи (по имени и `@username`). Агрегация — `panel/data.py`.
- **Рантайм вне git:** `roster.md`, `verified_users.json`, `admins.json`, `INDEX.md`, `embeddings_cache.json`, `*.jsonl`.
